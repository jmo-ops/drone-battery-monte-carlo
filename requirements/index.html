    Requirements

    This page captures the system/model requirements for the Drone Battery Monte Carlo model and how each requirement is evaluated.

    REQ-001 — Payload Weight Impact

    Requirement
    The model shall be able to determine the impact of battery draw on variable payload (0–5 lbs).

    Assumptions / Constraints - Drone follows a fixed course. - Payload varies between 0–5 lbs; all other variables held constant. - Assume 0 lbs payload for return trips in battery calculations.

    Question(s) Answered - How much does package weight impact battery draw?

    Primary Variable - Payload Weight

    Method - Baseline System Draw and Mass Penalty - Run X randomly generated runs with payload weight as the only variable to determine impact. - Monte Carlo analysis

    Outputs / Acceptance Evidence - Payload Weight impact in terms of battery draw (e.g., watts, Wh, or voltage draw — use whichever your model reports consistently)

    Validation Data Source - Battery Test Data Validation

    Analysis Deliverable - Plot/table of draw vs. payload weight (0–5 lbs), with summary statistics (mean, min/max, percentile bands)

    REQ-002 — Flight Distance Impact

    Requirement
    The model shall be able to determine total trip battery draw over variable distance (1–15 miles roundtrip).

    Assumptions / Constraints - Drone flies between 1 and 15 miles per trip (roundtrip). - Battery draw across all subsystems remains constant (other than distance-driven energy usage).

    Question(s) Answered - How many trips can the drone take on a single charge?

    Primary Variable - Flight Distance

    Method - Baseline System Draw and Mass Penalty - Run X randomly generated runs with flight distance as the only variable to determine impact. - Monte Carlo analysis

    Outputs / Acceptance Evidence - Flight Distance impact in terms of battery draw (e.g., Wh consumed per trip / voltage draw) - Estimated trips per charge across the range

    Validation Data Source - End-to-End Simulation

    Analysis Deliverable - Plot/table of draw vs. flight distance (1–15 mi RT), plus computed trips-per-charge bands

    REQ-003 — Trips Before Battery End-of-Life (EOL)

    Requirement
    The model shall be able to determine the total number of trips achievable for each candidate battery before end-of-life (EOL).

    Assumptions / Constraints - Drone flies between 1 and 15 miles per trip. - Battery performance degrades with recharge cycles per the candidate battery specification / life-cycle model.

    Question(s) Answered - Which candidate battery provides the greatest number of trips before EOL?

    Primary Variable - N/A (comparative evaluation across batteries using their specs and recharge/lifecycle model)

    Method - Battery specifications + number of recharges / lifecycle model - Run X randomly generated runs with different random inputs to determine which battery delivers the most trips - Monte Carlo analysis

    Outputs / Acceptance Evidence - Total number of trips before battery EOL (per candidate battery) - Ranked comparison + confidence bands

    Validation Data Source - Life Cycle Test Data

    Analysis Deliverable - Comparison chart/table per battery: expected trips-to-EOL, percentiles, and sensitivity notes
